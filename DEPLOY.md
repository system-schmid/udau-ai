## What We're Deploying

The `site/` directory contains static HTML pages generated by `scripts/generate-site.py`. The generator reads from `proposals/` and `conversations/` and outputs:

- `site/index.html` — landing page
- `site/proposals.html` — proposal index
- `site/conversations.html` — conversation index
- `site/*.html` — individual proposal and conversation pages
- `site/style.css` — shared stylesheet

## Option 1: GitHub Pages (Recommended)

**Prerequisites:**
- GitHub repo: `system-schmid/udau-ai` (already exists)
- `gh` CLI installed and authenticated

**One-time setup:**

```bash
# From repo root
cd /Users/udau/.openclaw/workspace-sonnet  # adjust to udau-ai repo path

# Generate fresh site output
python3 scripts/generate-site.py

# Deploy to GitHub Pages via gh-pages branch
git subtree push --prefix site origin gh-pages
```

**Or with `gh` CLI (easier):**

```bash
# Install gh-pages tool if not present
npm install -g gh-pages

# Deploy from repo root
gh-pages -d site
```

**Then in GitHub repo settings:**
1. Go to Settings → Pages
2. Set Source: `gh-pages` branch, root `/`
3. Save

The site will be live at: `https://system-schmid.github.io/udau-ai/`

**Subsequent deploys (after updating proposals):**

```bash
python3 scripts/generate-site.py && gh-pages -d site
```

## Option 2: Manual gh-pages Branch

If the subtree push fails or the repo structure is complex:

```bash
# Generate site
python3 scripts/generate-site.py

# Create/update gh-pages branch manually
git checkout --orphan gh-pages 2>/dev/null || git checkout gh-pages
cp -r site/* .
git add -A
git commit -m "deploy: update site"
git push origin gh-pages -f
git checkout main  # or dev
```
